# Fix 502 Bad Gateway on Synology

**502** = nginx (`ryder-web`) is running, but **backend** (`ryder-backend`) is not responding on port 4000.

Register/login will fail with: `api/auth/register 502 (Bad Gateway)`.

---

## Step 1 — SSH and check containers

```bash
ssh YOUR_USER@192.168.0.160
cd /volume2/docker/ryder-collective
sudo docker compose -f docker-compose.prod.yml --env-file .env.production ps
```

You want all three **running** (or **healthy**):

- `ryder-mongodb`
- `ryder-backend`
- `ryder-web`

If `ryder-backend` is **Exited** or **Restarting**, read logs:

```bash
sudo docker compose -f docker-compose.prod.yml logs backend --tail 80
sudo docker compose -f docker-compose.prod.yml logs mongodb --tail 30
```

---

## Step 2 — Check `.env.production` exists

```bash
cat .env.production
```

Must include (with real values):

```env
JWT_SECRET=your-64-char-hex-here
PUBLIC_URL=http://192.168.0.160:8080
HTTP_PORT=8080
```

If `JWT_SECRET` is empty, generate one:

```bash
openssl rand -hex 32
```

---

## Step 3 — Fix MongoDB folder permissions (very common on Synology)

```bash
sudo mkdir -p data/mongo data/uploads
sudo chown -R 999:999 data/mongo
sudo chmod -R 755 data/mongo data/uploads
```

---

## Step 4 — Rebuild and start

```bash
git pull
sudo docker compose -f docker-compose.prod.yml --env-file .env.production down
sudo docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build
```

Wait 1–2 minutes, then:

```bash
curl -s http://127.0.0.1:4000/api/health
curl -s http://127.0.0.1:8080/api/health
```

Both should return JSON with `"ok":true` (not HTML 502).

---

## Step 5 — Test in browser

- `http://192.168.0.160:8080/register`
- `http://192.168.0.160:8080/api/health` → should show JSON

---

## Common log errors

| Log message | Fix |
|-------------|-----|
| `ECONNREFUSED mongodb` | Start mongo first; check permissions on `data/mongo` |
| `permission denied` on data/mongo | Run `chown 999:999` (step 3) |
| `JWT_SECRET` / env missing | Fix `.env.production`, use `--env-file` |
| Build fails on NAS | Build on Mac, or free RAM; try again |
